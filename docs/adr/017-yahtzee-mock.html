Design notes for handoff

  • Tokens: no new top-level CSS variables. Yahtzee-only colors (--felt, --bakelite, --paper, --ink) are derived from the site palette and live in the game's local :root block.
  • Type: Fraunces with 'WONK' 1, 'SOFT' 50–100 for display moments; bumped to italic + WONK for the fanfare; Bricolage everywhere else. Already on the site — no extra font load.
  • Dice: 100% CSS + DOM (no SVG, no canvas). One .die shell, nine .pip positions selected by class. Faces are presentational only; data-face attribute will drive class toggling in JS. Built to scale via --d-size.
  • Held state: distinguished by (a) translateY(-10px) elevation, (b) gold inner-stroke ring, (c) gold outer glow, AND (d) a separate lock-icon SVG that sits above the die. Color-blind-safe by construction.
  • Tumble animation: not shown in this static mock. Plan: 3D rotateX/Y/Z CSS keyframe ~600ms with a settle bounce; held dice are excluded from the keyframe. prefers-reduced-motion swap shows the result instantly with a soft fade.
  • Scorecard glyphs: hand-drawn inline SVG per row (pip stacks for upper section, gable for full house, ascending step for straights, mountain + crown for Yahtzee, arc for Chance). Ink color = var(--ink), opacity 0.6 for unfilled rows.
  • Preview row: live score appears on focus/hover with a gold inset bar + "Preview" tag. Click commits; Esc cancels.
  • Yahtzee fanfare: three layered effects — animated radial burst (::before), slow-spinning conic-gradient ray pattern, floating glowing dice. Subtitle line shows the points awarded ("+50" first time, "+100" on each subsequent Yahtzee bonus) so the rule communicates itself. Add screen shake + horn sting in the implementation pass.
  • Pause overlay: backdrop-filter blur + saturate keeps the table visible but de-emphasizes it. Toggles for Music / SFX / Reduced motion are persisted to localStorage["yahtzee-audio"] and ["yahtzee-prefs"].
  • Hot-seat layout: the ADR calls for "side-by-side scorecards on desktop" — for >2 players the right rail becomes a tabbed scorecard with the active player's card shown by default (other tabs show name + running total in a strip). The mock shows the 1-card variant; multi-tab variant is a render flag, not a separate screen.
  • Mobile: below 720px the grid collapses; scorecard moves below dice; roll button stays in the thumb zone. Header turn-bar wraps to two lines if names are long.
  • Konami easter egg: swaps --bakelite for a deeper gold ramp on the next round only. Cosmetic; no scoring effect.
